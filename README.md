I build practical AI-assisted internal tools, workflow dashboards, and automation systems that solve boring problems before they become expensive ones.

Right now, a lot of my work sits at the intersection of:

## What I've built recently

### Mission Control
A Next.js operational dashboard for managing projects, tasks, approvals, docs, memory, activity, and AI-assisted workflow surfaces in one place.

Highlights:
- Real project/task data model tied to actual build history
- Control Center for editing live rule/config files
- Resume Workshop with lane-based resume routing and job analysis
- ContentBot surfaces for queueing, composition, analytics, and persona management
- Activity feed, approvals, docs, memory, and team views
- Local-first JSON-backed architecture instead of unnecessary backend sprawl

**Stack:** Next.js, React, TypeScript, Node.js

---

### OpenClaw Control Center
A lightweight local dashboard for monitoring OpenClaw without dragging in a bloated frontend stack.

Highlights:
- Status summary and recent activity
- Warning/error visibility
- Permissions state snapshot
- Editable local job-application answer bank
- Project visibility for local builds, paths, and URLs
- Graceful local-file fallbacks instead of breaking theatrically

**Stack:** Node.js, local JSON/data files, lightweight frontend

---

### ResumeBot / Job Search Systems
A set of tools and workflows for structured job search operations, resume routing, and safer application automation.

Highlights:
- Lane-based resume architecture (Implementation / Solutions / AI customer-facing)
- Resume analysis endpoint that scores job text against lane signals
- Resume Workshop UI with routing guide, intake, editor, and analysis panel
- LinkedIn automation scaffold using Selenium + Chrome remote debugging session reuse
- Clear stop conditions for weird flows, CAPTCHAs, and external ATS detours

**Focus:** workflow quality, routing logic, and practical automation over fake "AI magic"

---

### ContentBot
An AI-assisted publishing and content workflow system designed for structured output, persona control, and operational visibility.

Highlights:
- Unified Content Hub with queue + composer workflow
- Persona/profile system with platform-aware publishing settings
- Voice inspiration model for better output steering
- Analytics and content feed views
- Shared model-router / fallback architecture instead of fragile one-off prompt wiring

**Stack:** Next.js, React, TypeScript, structured JSON APIs

---

### QA Pipeline
A Playwright-based test suite for validating dashboard navigation and core workflows.

Highlights:
- Multi-suite regression coverage
- Smoke tests across major routes
- Coverage for navigation and content workflows
- Built to keep the dashboard from silently rotting, which is frankly a low bar but still apparently rare

**Stack:** Playwright, Node.js
